The Idea

What if software didn't need infrastructure?

RappterZoo is a gallery of 616 self-contained HTML apps — games, synthesizers, physics simulations, 3D worlds, creative tools — each one a single .html file with all CSS and JavaScript inline. Download any app and it works forever. No server required.

📦

One File = One App

Every app is a complete, self-contained HTML document. All CSS, JavaScript, and assets are inline. Right-click → Save As → it works forever.

✈️

Works Offline

Zero network requests. No CDNs, no API calls, no analytics. Open the file in a browser, even on a plane with no WiFi, and everything works.

💾

Local-First Data

All persistence uses localStorage. Your data never leaves your machine. Apps that manage user data include JSON import/export built in.

🔧

Zero Build Process

No webpack, no vite, no npm, no node_modules. The entire project is hand-editable static files served directly by GitHub Pages.

🧬

AI-Evolved

Apps improve through "molting" — an autonomous pipeline that scores, ranks, and iteratively evolves each app through generations. The best survive.

🌐

Truly Portable

No vendor lock-in. No accounts. No platform dependency. Every app is a standard HTML file that works in any browser on any OS, forever.

Under the Hood

How a single HTML file does all this

No magic, just discipline. Here's the anatomy of a RappterZoo app.

<!-- Every app is structured like this: --> <!DOCTYPE html> <html> <head> <title>My App</title> <meta name="viewport" ...> <!-- Embedded metadata for the gallery --> <meta name="rappterzoo:category" content="games_puzzles"> <meta name="rappterzoo:type" content="game"> <style> /* ALL CSS inline — no external stylesheets */ </style> </head> <body> <canvas id="game"></canvas> <script> // ALL JavaScript inline — no imports, no CDN // localStorage for saves, Web Audio for sound // Canvas 2D or WebGL for rendering const canvas = document.getElementById('game'); function gameLoop() { /* everything happens here */ } requestAnimationFrame(gameLoop); </script> </body> </html>
1

Write a single HTML file

All CSS in a <style> tag, all JS in a <script> tag. Canvas for graphics, Web Audio for sound, localStorage for saves. That's the entire stack.

2

Drop it in a category folder

Place your file in apps/games-puzzles/, apps/audio-music/, or any of the 9 categories. CI auto-sorts files dropped in the wrong place.

3

Register it in the manifest

Add a JSON entry to apps/manifest.json with the title, description, and tags. The gallery picks it up instantly.

4

The autonomous engine takes over

The Molter Engine scores your app across 6 quality dimensions, ranks it against the collection, and can iteratively evolve it through generations — each one improving a different aspect.

What Makes This Different

The apps evolve themselves

RappterZoo isn't just a gallery — it's an autonomous ecosystem. An AI pipeline continuously scores, ranks, and improves apps through a biological "molting" process.

🧬 Genetic Recombination

Top-performing apps donate "genes" (render pipeline, physics engine, audio, particle systems) that get crossbred to produce novel offspring.

🦎 Adaptive Molting

Each app's identity is analyzed — a synth gets better synth controls, a game gets better gameplay. The medium IS the message.

🏥 Runtime Verification

7 health checks catch apps that look good on paper but crash in a browser. Broken apps get penalized, healthy apps get boosted.

📊 Adaptive Ranking

Structural integrity, scale, craft sophistication, completeness, engagement potential, and polish — each scored and weighted by what the app IS.
