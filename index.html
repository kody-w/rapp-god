The map — which repo houses which part

How it works. Lens A fetches each canonical file (raw.githubusercontent.com) and this repo's snapshot, hashes both (SHA-256), compares. Lens B fetches two live files that must agree and compares their hashes. In sync = identical; drift = they differ. Re-sync the snapshots with bash sync.sh; fix cross-repo drift at the source. A GitHub Action (god-drift) runs the same checks on a schedule, so drift is caught even when no one's looking. Globally readable via GitHub Pages + raw.githubusercontent.com.
