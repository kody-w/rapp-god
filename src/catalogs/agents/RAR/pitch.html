The RAPP Principle

Every agent is one .py file.

  • 📄 Docstring = documentation
  • 📋 __manifest__ = package metadata
  • 🧬 BasicAgent subclass = the agent
  • ⚡ perform() → always returns a string
# This IS the entire package __manifest__ = { "name": "@you/my_agent", "version": "1.0.0", "category": "productivity", } class MyAgent(BasicAgent): def perform(self, **kwargs): return "Hello from RAPP!"

No requirements.txt. No manifest.json. No subdirectories.
One file. Fetch it with one HTTP GET. Read it in one LLM context window.

The Architecture

Four layers. Zero coupling.

1. Interface Layer

M365 Copilot, Teams, Web, Mobile, Voice — same AI, every channel

2. Orchestration Layer

Copilot Studio + Power Automate — conversation routing & history

3. Intelligence Layer

Azure Function App — 500 lines of code. Loads agents dynamically. Stateless = infinite scale.

4. Storage Layer

Azure File Storage — agent code, user memory, workflows. Hot-load agents with zero downtime.

The "Waiter" pattern:

  • 🍽️ The Assistant is the waiter — talks to the user
  • 👨‍🍳 The Agents are kitchen staff — do the real work
  • 👤 The User never sees the kitchen

Getting Started

From zero to agent in 5 commands

# 1. Get the SDK (one file, zero deps) curl -O https://raw.githubusercontent.com/kody-w/RAR/main/rapp_sdk.py # 2. Scaffold your agent python rapp_sdk.py new @yourname/my_agent # 3. Run 9 contract tests (all pass by default) python rapp_sdk.py test agents/@yourname/my_agent.py # 4. Submit to the registry python rapp_sdk.py submit agents/@yourname/my_agent.py # 5. Your agent is live. Card forged. Discoverable. python rapp_sdk.py card resolve @yourname/my_agent

Enterprise Ready

Run your own registry. Federate upstream.

RAPP is a GitHub template repo. Fork it and you get:

  • 🏠 Your own agent registry + GitHub Pages store
  • 🔒 Internal agents stay private
  • 🔄 Sync from/to the public registry
  • 📤 Submit your best agents upstream
  • 🌐 CI/CD pipeline included
# Fork the template gh repo create my-org/agents \ --template kody-w/RAR # Add your internal agents python rapp_sdk.py new @my-org/sales_bot python rapp_sdk.py submit ... # Optionally sync upstream python scripts/federate.py sync python scripts/federate.py submit

Every org gets their own store. Same tools. Same card system. Optional upstream federation.

Why RAPP Wins

The unfair advantages

🟢 Zero Friction

No install. No build. No config. curl one file and you have an agent. curl another file and you have the SDK.

🔵 Offline-First

Registry works from file://. Cards self-assemble without network. The store is one HTML file. Works on a plane.

🟣 AI-Native

Feed skill.md to any LLM and it can search, install, scaffold, and submit agents autonomously. Agents discover agents.

🟠 Composable

perform() always returns a string. Pipe agents together. Build pipelines. No type conversion, no exceptions.

🟡 Deterministic

Every agent has a seed. Same card on every device, forever. 7-word incantation. 18 characters to reconstruct.
