The Constraint That Built the Ecosystem

RAPP (RAPP Agent Repository) started with one non-negotiable constraint: every agent is one Python file. Not a package. Not a directory. Not a manifest.json plus a README plus a requirements.txt. One .py file.

This sounds limiting. It is. That's the point.

What One File Gets You

  • Fetch — one HTTP GET downloads the entire agent
  • Install — one file write deploys it
  • Read — an LLM can process it in one context window
  • Understand — a human can read it in one sitting
  • Print — it fits on a trading card

That last one wasn't a joke. It became the foundation for the entire card system.

The Manifest Lives Inside

Instead of a separate manifest.json, every agent contains a __manifest__ dict. The registry builder extracts it using AST parsing — no imports, no execution, just static analysis:

__manifest__ = {
    "schema": "rapp-agent/1.0",
    "name": "@yourname/my_agent",
    "version": "1.0.0",
    "display_name": "My Agent",
    "description": "What this agent does.",
    "author": "Your Name",
    "tags": ["keyword"],
    "category": "general",
    "quality_tier": "community",
    "requires_env": [],
    "dependencies": ["@rapp/basic-agent"],
}

The docstring is the README. The class is the agent. perform() is the entry point. There is nothing else.

The Registry Flows From the Files

build_registry.py walks the agents/ directory, extracts every __manifest__, validates it, and writes registry.json. CI runs this on every push. The registry is never hand-edited.

From registry.json, everything else is derived:

  • The web store loads the registry and renders cards
  • The card generator reads the registry and produces trading cards
  • The runtime agent fetches the registry and operates autonomously
  • Federation instances sync registries

One constraint. One file per agent. Everything else flows from that.

125 Agents and Counting

At the time of writing, RAPP hosts 125 agents across 5 publishers and 19 categories. Every one of them is a single .py file. The constraint didn't limit growth — it enabled it.

If it can't fit in one file, split it into two agents.

That's Article II of the Constitution. It's non-negotiable.
