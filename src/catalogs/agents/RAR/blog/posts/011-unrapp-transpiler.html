The Narrow Subset That's Actually Wide

RAPP agents are Python. The browser runs JavaScript. The conventional answer is "run Python in WASM" (Pyodide, ~15MB download, 3-second cold start). Our answer: don't run Python at all. Transpile the narrow subset that agents actually use.

We analyzed 128 agents and found they share a surprisingly small surface area:

  • Module-level data dicts (UPPER_CASE = {...})
  • A perform() dispatcher
  • Methods that format data into strings
  • f-strings, list comprehensions, .get(), sorted(), .join()

No imports that matter. No file I/O. No network calls (the synthetic data is inline). No classes with inheritance. The agents are pure functions over static data. That's transpilable.

The Five Stages

UnRAPP is a 5-stage Python-to-JavaScript transpiler. Each stage handles one concern:

Stage 1: Preprocess. Strip everything the browser doesn't need: imports, sys.path, class wrappers, __init__, super().__init__, if __name__. The __manifest__ dict is parsed separately. What's left is data and methods.

Stage 2: Extract Data. Find UPPER_CASE = { or UPPER_CASE = [ assignments at module level. A bracket-counting parser extracts the full literal (handling nested dicts, trailing commas, Python booleans). Convert to JavaScript objects.

Stage 3: Extract Dispatch. Find the perform method. Parse the dispatch pattern — either a {"op": self._method} dict or an if op == "x" / elif chain. Output the list of operations.

The Hard Parts

The transpiler's complexity lives in expression parsing. Python's syntax has ambiguities that require depth-aware parsing:

Nested bracket access. TIER_STRUCTURE[m["tier"]] — a regex can't tell where the outer brackets end. We wrote splitBracketAccess() with full bracket-depth tracking.

Ternary inside arithmetic. rating + (10 if in_zone else 0) — the if/else keywords appear inside a sub-expression. splitTernary() only matches at bracket depth zero.

Chained .get(). data.get("key", {}).get("sub", 0) — each .get() becomes PY.get(), but the object of the second call includes the first. A 3-pass rewriter handles arbitrary nesting.

The in operator vs. dict access. c in t["certifications"] — the dict-chain handler tried to match before the in operator. Order of operations in the parser matters: check in/not in before dict chain access.

What It Doesn't Handle

UnRAPP is deliberately narrow. It won't transpile:

  • Agents that import external libraries
  • Agents that make network calls (use the chat mode's built-in agents for that)
  • Complex class hierarchies or metaclasses
  • Generator expressions used as iterators (only as arguments to sum(), any(), all())

This is a feature. The transpiler's boundary defines what "runs in the browser" means. If an agent doesn't fit, it needs the real brainstem. The registry's requires_env field tells you which is which.

The Result

128 agents, ~4ms average transpile time, zero external dependencies. The transpiler is 800 lines of JavaScript doing what would normally require a Python AST library, a code generator, and a bundler. Instead, it's string manipulation in a <script> tag. The single-file principle applies to the tools, not just the agents.
