The Problem With Controlled Fusion

The companion card needed to be special. Not just "your favorite card" but something uniquely yours — something that couldn't be manually engineered or gamed. The inspiration came from emergent systems: what if the card's visual identity was determined by factors you don't control?

Three Inputs, No Controls

The Forge combines three values:

InputSourceYou Control It?
Agent IdentityHash of the companion agent's nameYou chose the agent, but not its hash
Owner IdentityYour cryptographic fingerprint (if signed) or a persistent session seedNo — derived from your signing key or randomized on first visit
Time EpochCurrent 15-minute time window: Math.floor(Date.now() / (15 * 60 * 1000))No — shifts automatically

These three values are concatenated and hashed:

function forgeHash(agentName, ownerSeed, timeSeed) {
  const raw = agentName + '|' + ownerSeed + '|' + timeSeed;
  return Math.abs(Array.from(raw).reduce(
    (acc, c) => ((acc << 5) - acc + c.charCodeAt(0)) | 0, 0
  ));
}

The forge hash drives everything: ASCII art parts, stat distributions, and the companion's visual identity. Every 15 minutes, the time epoch shifts and the card transforms.

What the Forge Produces

  • Forged ASCII Art — same procedural system as regular cards, but seeded with the forge hash instead of the agent name hash. Different body parts, different character.
  • Forged Stats — POWER, SPEED, LOGIC, CHAOS, GUARD are recalculated from the forge hash. The ranges shift (15-94% instead of 20-94%). Every forge cycle produces different stats.
  • Purple Styling — forged cards get a purple border, purple-colored art, and a "FORGED" label to distinguish them from base cards.
  • Bond Countdown — the deck bar shows a countdown to the next forge shift: "Bond shifts in 7m".

Why Emergence Matters

If users could control the forge output, they'd optimize for the "best" card. The forge prevents this by design:

  • You can't choose when the epoch shifts
  • You can't change your fingerprint
  • You can't predict what the hash will produce
  • The same companion + same owner at a different time = different card

This is family-friendly emergence. No manual engineering. The card emerges from the intersection of who you are, what agent you chose, and when. Every forged card is a snapshot of that moment.

Implementation Details

The forge integrates into the existing card rendering pipeline:

  1. agentToCard() checks if the current agent is the deck companion via getCompanion()
  2. If forged: uses forgeStats() and forgeAsciiArt() instead of the base functions
  3. The modal ASCII peek also detects forge state and shows "FORGED Card" in the collapsible details
  4. A 60-second interval refreshes the bond countdown timer
  5. Owner identity prefers the cryptographic fingerprint from the signing system, falling back to a localStorage session seed

Field Note

The key design constraint: "the forge should not be possible for the person to manually engineer." This ruled out any UI that lets users pick parts, choose colors, or influence the output. The forge just happens. You watch. That's the point.
