Five Systems That Had to Die

Before the simplicity audit, RAPP's store had five independent complexity systems. Each one sounded reasonable in isolation. Together, they created an interface that required a manual.

What We Killed

SystemWhat It DidWhy It Died
Tier GatingHid tabs and features based on user "tier" (1-3)Users had to earn features they could already see existed
Wave UnlocksProgressively revealed UI sections after actionsCreated an invisible queue users couldn't predict
Intent ToggleSwitched between "explore" and "deploy" modes, reweighting search resultsRequired users to understand a new concept ("intent")
Mode DialRotated between interface complexity levelsThree modes = three products to learn
Quality Seal TabSeparate tab explaining what tier badges meantIf badges need explanation, the badges are wrong

The Principle

Don't make users learn concepts. If a feature requires explanation, rename it or remove it. No jargon. No gating. No progressive disclosure that hides useful things behind walls.

This became the governing design principle, written directly into the Constitution (Article IX).

What Replaced It

Nothing. That's the point.

  • All tabs visible — Browse, Leaderboard, Cards, Packs, Stream, Workbench, Submit. Always.
  • No mode dial — one interface for everyone.
  • No intent toggle — search returns results. Period.
  • No tier gating — _uiTier is hardcoded to 2 (max). Every feature unlocked.
  • No Roadmap tab — the roadmap is the git log.

The Cleanup

Removing five systems from a 500KB+ single-file app required careful surgery:

  • Stub functions that previously gated UI: setUITier(), applyUITier(), intentWeight() — all now return constants
  • CSS classes like .rar-guard, .wave-2, .tier-3 — removed from elements
  • Navigation buttons for Quality Seal and Roadmap — deleted
  • Mode dial button and its handler — deleted
  • Intent button and sort reweighting — deleted

The sort comparators in renderAgents() lost their intentWeight() terms. The header actions lost their dial button. The nav lost two tabs. The codebase got simpler and the product got better.

Field Note

The user's exact words: "I don't need to train users on what a 'hail' is." That line killed the hail button, and the principle behind it killed everything else. Complexity is easy to add and hard to remove. Remove it early.
