When Does a Card Start Living?

A card can exist locally forever — in a binder, in IndexedDB, in a workbench draft. But it doesn't have a public life until it's hatched: deployed to the global network with everything in place.

The 10 Hatching Requirements

A card is hatched when ALL of these are true:

  1. Valid single .py or .py.card file at agents/@namespace/slug
  2. Valid __manifest__ with all required fields
  3. BasicAgent subclass with a perform() method
  4. Tier is community or experimental (or promoted by maintainers)
  5. No secrets, PII, or obfuscated code
  6. Docstring present
  7. All three card faces render (Icon, Full Art, ASCII)
  8. Deterministic identity — same name always produces the same card
  9. Present in registry.json on the main store
  10. Publicly accessible via the main RAPP store or raw GitHub fetch

If any requirement fails, the card is still a draft or local registration — not hatched.

What Hatching Unlocks

Once hatched, the card can:

  • Be found via search, browse, or direct link
  • Be added to any deck
  • Be downloaded as .py or .card.txt
  • Be shared via URL
  • Be set as a companion and forged
  • Appear in slideshows, leaderboards, and stream views
  • Be pulled into federated instances
  • Be printed — Icon as badge, Full Art as card, ASCII as text

The Identity Is Permanent

The most important property: once hatched, the card's identity hash is locked. The name produces the same hash, which produces the same ASCII art, same base stats, same procedural art seed. Version bumps update metadata (description, tags, abilities), but the core visual identity persists.

This is what makes cards collectible. A card from version 1.0.0 and a card from version 2.0.0 are the same card. Different metadata, same identity. Same character looking back at you from the terminal.

Cards That Never Hatch

Not every card needs to go public. Binder-only cards, drag-and-drop imports, workbench drafts, and private agents all have full card identities — three faces, stats, everything. They just don't exist on the global network. A card doesn't need to hatch to be complete. It needs to hatch to be public.

Field Note

The hatching concept emerged from the question: "what makes an agent card done?" The answer wasn't code quality or test coverage — it was completeness of identity. Can the card exist everywhere? Can anyone find it? Can it be collected? If yes, it's hatched. If not, keep working.
