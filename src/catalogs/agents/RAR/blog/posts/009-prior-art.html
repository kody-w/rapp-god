The Inventions

These are the discrete innovations in the RAPP ecosystem. Each one is documented in the codebase, the Constitution, and the git history.

1. Single-File Agent Packaging

An AI agent is one .py file. The manifest (__manifest__) is embedded in the file via a Python dict, extracted by AST parsing with zero code execution. The docstring is the documentation. The class is the agent. One HTTP GET to fetch. One file write to install. One context window to read.

First committed: 30e0bd1, April 3, 2026, 16:15 ET

2. Agent-as-Trading-Card Identity System

Every AI agent is represented as a collectible trading card with stats (power, toughness, abilities), visual art, rarity tier, flavor text, and a deterministic identity hash. Cards are generated procedurally from agent metadata using a seeded PRNG — same agent always produces the same card. Cards can be collected into decks, traded via URL, graded, valued, and displayed.

First committed: 1ed07b4, April 3, 2026, 16:51 ET

3. Three Universal Card Faces

Every card renders in three modes: Icon (compact info card), Full Art (holographic trading card with parallax depth), and ASCII (pure monospace terminal card). All three faces are generated from the same data with zero external assets. The ASCII face works in any interface that supports text — terminals, CLI tools, air-gapped systems, plain-text printouts.

First committed: a29b640 (dual mode), April 3, 2026, 17:21 ET. ASCII face added April 4, 2026.

4. Deterministic Procedural Art Engine

A seeded PRNG generates unique SVG art for every agent. Seven visual styles. Semantic glyph compositions mapped to categories. Artist-override system for hand-crafted cards. Mulberry32 hash function for deterministic randomization. Same name = same art, always.

First committed: 99ec5f7, April 3, 2026, 20:21 ET

5. Local-First Agent Store

A zero-install, single-file web store (index.html) that runs entirely in the browser. Agents loaded from a JSON registry or dragged in from the local filesystem. Stored in IndexedDB. No server, no account, no upload. Works offline. Works air-gapped.

First committed: faf0986, April 3, 2026, 16:31 ET

6. Pack-Based Card Minting

Agents minted as collectible cards via pack openings with weighted rarity (common, uncommon, rare, mythic). Serial numbers, first-edition tags, and grading system (centering, edges, surface, corners). Card values estimated from rarity, grade, mint number, and community votes.

First committed: ca25e78, April 3, 2026, 19:19 ET

7. Agent Card Shell Format (.py.card)

A file format that wraps a bare agent (.py) with trading card metadata (__card__ dict). The card shell contains name, title, mana cost, colors, type line, rarity, power/toughness, abilities, flavor text, SVG art, set code, and artist. Shedding strips the shell to produce a bare agent. Re-shelling adds it back from the registry.

First committed: c4e2fa7 (v1), April 3, 2026, 22:17 ET. v2 with __card__ dict: April 4, 2026.

8. Deck Extension Chain (.py.card.DeckName)

File extension encodes full packaging hierarchy: .py (bare agent) -> .py.card (agent + card shell) -> .py.card.DeckName (agent + card + deck membership). Each layer is additive and reversible. Deck import/export by copying files. Runtime hotloading: switch active agent deck without restart, like switching AI models.

Defined: April 4, 2026.

9. Emergent Companion Forge

A companion card system where the selected companion's ASCII identity is fused with the owner's cryptographic fingerprint and the current 15-minute time epoch. Three inputs, no user controls. The forge output cannot be manually engineered. The card shifts every 15 minutes. Each forge instance is unique to the combination of who you are, what you chose, and when.

Defined: April 4, 2026.

10. Federation Protocol

RAPP instances cloned from the template operate as independent agent stores. Federation allows submitting agents upstream, syncing agent catalogs, and maintaining independent tier hierarchies. Issues-as-API pattern: submissions are GitHub Issues with JSON payloads, processed by CI.

First committed: April 3, 2026 (Constitution).

11. Local-First Agent Binder

Any user can clone RAPP and run their own local agent store. build_registry.py + generate_holo_cards.py + open index.html. That's it. Full browse, cards, decks, workbench, companion system. No server. No account. Optionally syncs with the main store.

Defined: April 4, 2026 (Constitution Article XVI).

12. Unified SVG Glyph Art System

Hand-crafted SVG glyph functions that render at any size. Two libraries: SEMANTIC_GLYPHS for card art composition, UI_GLYPHS for interface icons. Both rendered via svgIcon(name, color, size). No emojis anywhere. Consistent visual identity across all surfaces: cards, UI, pack art, badges, and print.

Defined: April 4, 2026.
