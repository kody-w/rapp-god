87 Emojis Found. 87 Emojis Killed.

A thorough audit of the RAPP codebase found 87 emoji usages: 21 Unicode emoji characters and 66 HTML entity emojis. They were everywhere — category icons, pack art, section headers, modal buttons, slab badges, quiz results, the cart, the search bar.

They all had to go.

Why Emojis Are Wrong for RAPP

Emojis look different on every platform. An emoji that looks polished on macOS looks completely different on Windows, Android, and Linux. They're rendered by the OS, not by us. We have zero control over their visual identity.

RAPP already had a hand-crafted SVG glyph system — SEMANTIC_GLYPHS — used by the card art generator. These glyphs are consistent, scalable, and part of the same visual language as the trading cards. Using emojis alongside them was a visual contradiction.

The Solution: Two Glyph Libraries

LibraryPurposeGlyphs
SEMANTIC_GLYPHSCard art composition (existing)brain, gear, code, wrench, pipeline, person, email, api, cloud, chart, shield, heart, money, factory, lightning, building, document, arrow, database, memory
UI_GLYPHSUI icons (new)search, cart, globe, pencil, trash, link, trophy, confetti, book, crown, crystal, flask, flame, speech, memo, bag, handshake, desktop, sparkle

Both libraries share the same rendering signature: (primaryColor, centerX, centerY) => svgContent. Both are rendered via the same svgIcon() function:

function svgIcon(name, color, size) {
  const fn = UI_GLYPHS[name] || SEMANTIC_GLYPHS[name];
  if (!fn) return '';
  const s = size || 20;
  return `<svg width="${s}" height="${s}" viewBox="0 0 ${s} ${s}"
    style="display:inline-block;vertical-align:middle">
    ${fn(color || '#fff', s/2, s/2)}</svg>`;
}

What Changed

  • Category icons — CAT_TO_EMOJI (dead code) replaced with CAT_TO_GLYPH mapping categories to glyph names
  • Pack icons — crystal ball / test tube / fire emojis replaced with crystal, flask, flame SVG art rendered at 48px
  • Cart — shopping bag emoji replaced with cart SVG glyph
  • Search — magnifying glass emoji replaced with search SVG
  • Section headers — speech bubble, document, memo emojis replaced with matching SVGs
  • Modal actions — trash, link, lightning, pencil, cart emojis all replaced
  • Slab viewer — trophy and money bag emojis replaced
  • Quiz results — trophy, confetti, book emojis replaced
  • Trophy case — crown emoji replaced

What We Kept

Functional typographic symbols were left alone:

  • Checkmarks, X marks, stars (rating system)
  • Up/down triangles (voting)
  • Left/right arrows (navigation)
  • Warning signs (status indicators)

These aren't emojis — they're Unicode typographic symbols that render identically everywhere. They serve a functional purpose, not a decorative one.

The Rule

No emojis. If you need a visual, use svgIcon(). If it doesn't exist yet, add it to UI_GLYPHS.

The visual identity is now unified: cards, UI, pack art, and badges all use the same SVG art system. No more platform-dependent rendering. No more visual contradictions.
