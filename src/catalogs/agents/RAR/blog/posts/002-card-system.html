The Problem: Cards That Can't Go Everywhere

The original agent cards were beautiful. Full Art holographic trading cards with parallax depth, prismatic refraction, and specular shimmer. They looked incredible on a screen.

But they couldn't exist in a terminal. They couldn't be printed as plain text. They couldn't travel through a CLI pipeline or render on an air-gapped system. They were screen-only.

The insight that changed everything came from a collaborator: "every card should have each of these so it can exist ANYWHERE in space and time, virtual or real."

Three Faces, One Identity

All three faces are generated from the same data: the agent's __manifest__ and a deterministic identity hash derived from the agent's name.

Procedural ASCII Art

The ASCII face needed unique characters per agent — not one generic robot per category. We built a procedural generation system with 8 variants for each of 6 body parts:

const ASCII_PARTS = {
  heads:   ['  .---.', '  /===\\', '  {===}', ...],
  eyes:    [' |o  o|', ' |*  *|', ' |@  @|', ...],
  mouths:  [' | -- |', ' | \\/ |', ' | .. |', ...],
  bodies:  [' |####|', ' |/\\/\\|', ' |++++|', ...],
  legs:    ['  /||\\', '  /  \\',  '  |  |', ...],
  bases:   [' /    \\', ' ======', ' ------', ...],
};

Each part is selected using a hash of the agent name: 8^6 = 262,144 possible unique characters. Same agent name, same character, always.

The Card.txt Download

Any card can be downloaded as a .card.txt file — a box-drawn ASCII card with name, tier, art, stats, flavor text, and footer. Pure text. Works everywhere:

The Companion System

Each deck gets one companion card — a single agent that represents the deck's identity. The companion is marked with a star badge and shown in the deck bar. When the companion is set, the Forge takes over its ASCII face (see the Forge post for details).

One companion per deck. Like a signature agent. Persisted in localStorage, keyed by deck name.

Why It Matters

A card that only works on screens is a decoration. A card that works in a terminal, on paper, in a CLI tool, and on a phone is a universal identity. The agent goes everywhere its card goes.
