# RAPP + X™

**RAPP + X™** names the RAPP ecosystem's headless collaboration pattern:

> One `/chat`-shaped capability contract where `X` may be a person, AI,
> digital twin, rapplication, Brainstem, or neighborhood peer.

The UI is an optional client. Every participant uses the same capability
surface, so implementations may differ behind the wire without requiring a
human-only path, agent-only path, or translation layer.

## Mark usage

- Use **RAPP + X™** on first prominent use.
- Plain **RAPP + X** is acceptable in repeated technical prose and code.
- Do not use **RAPP + X®** unless registration has been granted in the relevant
  jurisdiction.
- Do not describe GitHub identity or a RAR receipt as a cryptographic publisher
  signature. A RAR notarization receipt binds canonical `sha256-lf-v1` bytes to
  named checks.

## Legal handoff checklist

This repository does not itself register a trademark. Before filing, the mark
owner and qualified trademark counsel should:

1. Confirm the legal owner and jurisdictions.
2. Run clearance searches for confusingly similar marks.
3. Select the applicable goods/services classes.
4. Preserve dated specimens showing actual use.
5. File and track applications with the relevant trademark offices.
6. Define licensing, quality-control, enforcement, renewal, and assignment
   policies for the open RAPP ecosystem.
