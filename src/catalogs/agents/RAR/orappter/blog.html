Introducing ORAPPTER Playground
launch playground

ORAPPTER Playground is now live — a complete local-first AI runtime that runs entirely in your browser. No download, no install, no server, no API key. Just open the page and start chatting with a real language model running on your GPU.

ORAPPTER Playground is now live — a complete local-first AI runtime that runs entirely in your browser. No download, no install, no server, no API key required. Just open the page and start talking to an AI that runs on your hardware.

Powered by WebLLM and WebGPU, the Playground achieves native GPU inference directly in your browser tab. The default model is Gemma 4 (or the best available Gemma variant from the WebLLM model library), giving you a capable, conversational AI out of the box.

What's included

  • A full LisPy REPL built in — the same API your agents use to interact with models
  • 130+ models available from the WebLLM model library
  • Pull, run, and chat with models — the same workflow as Ollama, but in a browser tab
  • Agent card activation — bring any Rappter agent card to life as an AI persona
  • Knowledge base and conversation memory that persists across sessions

How it works

WebLLM compiles open models to WebGPU shaders using the MLC compilation pipeline. Model weights are cached in IndexedDB after the first download, so subsequent loads are near-instant. All inference runs entirely in-browser — your data never leaves your device.

How ORAPPTER Works: WebGPU + LisPy
technical architecture

Under the hood, ORAPPTER routes agent code through a clean abstraction layer: your LisPy scripts call the ORAPPTER API, which dispatches to whichever backend is available — WebLLM in the browser or Ollama on the command line. Same code, different runtimes.

ORAPPTER's architecture is built around a single idea: agent code should run the same everywhere. Your .lispy scripts call the ORAPPTER API, which dispatches to whichever backend is available — WebLLM in the browser, or Ollama on the CLI.

The stack

Agent code (.lispy) → ORAPPTER API → Backend adapter
                                        ├── WebLLM (browser)
                                        └── Ollama  (CLI)

Why LisPy?

LisPy is homoiconic, async-capable, and runs identically in both browser and CLI environments. That makes it a natural fit for agent scripting — you can prototype in the Playground and deploy to a server without changing a line of code.

WebGPU inference

Models are compiled to WASM + WebGPU shaders by the MLC toolchain, running at near-native speed on your GPU. The compilation happens once; after that, cached artifacts make subsequent loads almost instant.

Offline-first layers

  • Layer 0 — Deterministic cards: pure math, no AI needed
  • Layer 1 — Template responses: rule-based fallbacks
  • Layer 2 — Local LLM: full generative AI via WebLLM or Ollama

Model caching

Model weights are downloaded once and stored in the browser's Cache API and IndexedDB. They persist across sessions and browser restarts, so you only pay the download cost once per model.

What's next

We're working on service worker support for full offline capability, WebNN as a CPU fallback for devices without WebGPU, and support for larger Gemma 4 model variants as WebGPU memory limits improve.

The Agent Card System
agents cards

Every agent in the Rappter ecosystem has a trading card identity — a deterministic, collectible representation generated from its name. Cards aren't just cosmetic: activate one and it becomes a live AI persona powered by your local model.

Every agent in the Rappter ecosystem has a trading card identity. These aren't random — they're deterministic. The same agent name produces the same card everywhere, forever, using a seeded PRNG. No server, no database, no coordination required.

Card anatomy

  • Display name — the agent's public identity
  • Type line — what kind of agent it is
  • Rarity — common, uncommon, rare, legendary
  • Power / Toughness — numeric stats for battles
  • Flavor text — a thematic one-liner

Cards as AI personas

Activating a card turns it into a live conversational agent. The card's attributes — name, type, flavor text — become the system prompt. Your local model takes on the card's personality and responds in character.

(activate-card "@borg/card_smith_agent")

This turns CardSmith into a live AI persona you can chat with, powered entirely by your local model.

Card battles

Two activated cards can answer the same question side by side. You read both responses and decide the winner. It's a fun way to compare agent personalities and test prompt engineering.

The binder

Your binder is your portfolio of collected agent cards. Browse them, sort by rarity, and activate any card to start a conversation. The entire binder works offline — deterministic generation requires no AI and no internet connection.

Cards + ORAPPTER

With ORAPPTER, cards come to life. What was previously a static collectible becomes a real conversational AI agent backed by whatever local model you have loaded. The card system is the bridge between the Rappter agent registry and the ORAPPTER runtime.
