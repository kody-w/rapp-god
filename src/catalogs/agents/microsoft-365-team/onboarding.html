The 4-step setup

  1. Get a brainstem running locally

    One-time install. If you have internet:

    curl -fsSL https://kody-w.github.io/RAPP/installer/install.sh | bash

    If you're fully airgapped, you'll need a copy of kody-w/RAPP on disk too — see OFFLINE_SETUP.md.

  2. Drop one agent into your brainstem

    Copy the bootstrap or the joiner (depending on path) into your brainstem's agents/ directory:

    # Path A — online via GitHub
cp ./agents/egg_hatcher_agent.py ~/.brainstem/agents/

# Path B — offline via .egg
cp ./microsoft-365-team_offline_bootstrap_agent.py ~/.brainstem/agents/

    The brainstem hot-reloads on the next chat request — no restart needed.

  3. Tell your brainstem to set you up

    # Path A
join <owner>/<this-repo-name>

# Path B
EggHatcher egg_path=/path/to/<neighborhood>.egg

    The agent verifies sha256 on every workflow file in rar/index.json, installs them into your local agents/, and records the subscription. Refuses on hash mismatch.

  4. Mint your per-handle workspace

    SesWorkspaceInit setup

    Creates your front door at ses/<your-handle>/ in the workspace (sanitized — no customer data) AND your local-only customer data dir at ~/.microsoft-365-team-data/<your-handle>/customers/. Pass handle=alice if you're offline (no gh auth).

You're done. Your brainstem now has 9 this neighborhood agents loaded. Try: Twin intro for an orientation in the owner's voice, or EngagementFactory slug=hello-world customer_name="Test Co" ask="..." to start your first engagement.
