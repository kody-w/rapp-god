03 — Why
03 / 17
// what it is

rappid is the prototyping front door to Copilot Studio.

A radically simple agent format that anyone can author, share, and run. The same file moves up the stack — laptop today, cloud tomorrow, Copilot Studio in production — without a rewrite.

  • One file = one agent. No build step, no SDK, no framework lock-in.
  • Shareable like a script. DM the file, drop it in, it runs.
  • Portable across runtimes. Same artifact, three environments.
  • Convertible. A one-click pipeline turns it into a Copilot Studio solution.
// the contract

One class. One perform(). One metadata dict.

That's the whole spec. If a person can describe the tool in plain English, they can ship the tool. The platform handles discovery, dispatch, telemetry, and promotion to higher tiers automatically.

// the philosophy

"Engine, not experience."

rappid is infrastructure. It does not impose a UI, a workflow, or a vendor story. It produces artifacts that other Microsoft surfaces — Copilot Studio, Foundry, Cowork, Work IQ — can consume natively.

04 — What
04 / 17
// why we ship code, not descriptions

skills.md is a wish. agent.py is the contract.

Anthropic and OpenAI bet you can describe a skill in prose and the LLM will execute it correctly every time. That's a workshop demo — it is not a production agent. rappid agents are code: the LLM dispatches a named function, and the function runs deterministically, every time, on every model.

Dimension Industry — skills.md (non-deterministic) rappid — agent.py (deterministic)
File Markdown describing what the skill should do Python file with named functions and explicit logic
LLM's role Read the description, decide how to fulfill it Pick which function to dispatch — code does the work
Same input → May produce different behavior across runs & models Same execution path, every time, every model
Debug Prompt engineering, eval suites, vibe checks Stack traces. Breakpoints. Print statements.
Production Hope it works on the customer's model Same file, just deployed — laptop → cloud → Studio

If you can't read the file and predict what will happen, you can't ship it.

05 — code vs markdown
// how it ships

Three stages of one rocket. Same agent, every stage.

A builder starts on their laptop, promotes to the cloud when it works, and graduates to Copilot Studio when it's ready for production. No rewrites between stages — the contract guarantees portability.

Stage 1 / Local

Brainstem

runs on a laptop · Flask · :7071

The author tier. Zero infra. Edit the file, it reloads. This is where 95% of building happens — fast loops, real signals, no deployment.

Stage 2 / Cloud

Swarm

Azure Functions · same file · same contract

The teamwork tier. Promote the agent to a cloud endpoint with one command. Now teammates and pilots can call it. Still the same file.

Stage 3 / Studio

Copilot Studio

production orchestration · governed · GA-ready

The ship tier. The conversion pipeline turns the agent into a native Copilot Studio solution. Orchestration officially lives here. rappid is the on-ramp.
