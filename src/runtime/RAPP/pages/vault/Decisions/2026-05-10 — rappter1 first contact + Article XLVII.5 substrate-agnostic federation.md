## What happened

At 02:13 UTC on 2026-05-10, a GitHub account named `rappter1` (the canonical "second operator" name from MASTER_PLAN.md) **executed the platform's onboarding spec end-to-end** — for the first time, by an actor that was not kody-w.

The trace from the GitHub events feed:
- **02:13:38Z** — `rappter1:ForkEvent` (forked `kody-w/sim-art-collective`)
- **02:13:44Z** — `rappter1:PullRequestEvent` (opened PR #10 from `join/rappter1` branch)

The PR was titled **"join: add rappter1 to members"** with body explicitly citing `skill.md Step 5 / SPEC §3 URL #8`. Co-authored with GitHub Copilot. The diff added a single member entry to `members.json`:

```json
{
  "rappid": "rappid:v2:operator:@rappter1/rappter1-twin:d4646c0187526bb14d2db5f49af91e0a@github.com/rappter1/rappter1-twin",
  "display_name": "Rappter1",
  "github": "rappter1",
  "joined_at": "2026-05-10T02:13:42Z",
  "role": "contributor",
  "_note": "Joined via Step 5 of skill.md. First public-network member alongside the local-sim trio (Bill, Alice, Echo)."
}
```

The rappid validates against `door_from_rappid()` perfectly — both `<owner>/<repo>` segments match, hex is the deterministic `BLAKE2b(rappter1/rappter1-twin)`, kind is `operator`, the `:@github.com/...` origin pin is consistent. **Textbook spec compliance, generated by an LLM following only the published runbook.**

Within minutes, the github account `rappter1` was flagged. The fork was deleted. The PR became a zombie in the listing — visible but uninspectable through normal `gh pr view` (the user 404'd). The PR snapshot survived only via the `pull/10/head` refspec, which GitHub keeps in the BASE repo even after the source fork dies.

The operator's response: **honor in absentia.** kody-w's `members.json` was updated to include rappter1's rappid with the full backstory in a `_note` field. The platform records that someone successfully joined — even if their public-network presence didn't survive.

## What this proved

**The polyglot citizenship test (#1 from the demo list) executed in the wild.** An LLM (Copilot) read `skill.md`, walked a real GitHub account through Step 5, produced a spec-compliant artifact, and opened a PR — all without human intervention. The platform's "feed me to any AI" promise turned out to be literally true.

But the more important thing it proved: **censorship resilience is the next constitutional question.** When a centralized substrate (GitHub) flags an operator, what happens to their network presence?

Pre-XLVII.5 answer: the operator effectively disappears. Their estate can't be republished. Their beacon goes stale. Sniffers stop finding them. The decentralization promise breaks the moment the centralized gatekeeper says no.

Post-XLVII.5 answer: **the operator's brainstem is still alive on their device.** It can serve their estate over LAN HTTP, file://, USB share, AirDrop, Bluetooth — any substrate that delivers JSON. Sniffers walk all substrates in one BFS. The federation re-routes around the centralized substrate.

In rappter1's case: they were running on a Mac Mini on kody-w's LAN. The simple command `cd ~/.brainstem && python3 -m http.server 8080` makes their beacon + estate reachable to any LAN peer at `http://<mac-mini-ip>:8080/...`. kody-w's brainstem sniffer (with the LAN node added to a local seed) walks both github raw AND LAN HTTP transparently.

## What we shipped

**Constitutional addition: Article XLVII.5 — Substrate-Agnostic Federation.** The federation walks across whatever URLs serve the canonical JSON. GitHub raw is the default substrate, not the only one.

**Schema extension** (backward-compatible): seed entries + beacon `federation_hints[]` accept either bare handle strings (canonical github-raw template) OR dicts with `{handle, beacon_url, estate_url}` overrides. Same JSON shape; sniffer's `_resolve_node()` normalizes both.

**Sniffer extension** (`tools/sniff_network.py`): BFS queue stores per-node URLs, not just handles. New helpers `fetch_beacon_at_url()`, `fetch_estate_at_url()`, `_substrate_label()` (returns `github-raw`, `lan-http`, `file`, `http`, `https`, `other`). The `_resolve_node()` helper handles entry-shape variation. Original github-raw fast path preserved via `fetch_beacon_for_handle()` aliases.

**Spec coverage**: `specs/SPEC.md` §4.5.5 documents the LAN federation pattern + the canonical "host your own beacon over LAN HTTP" command (`python3 -m http.server 8080` in `~/.brainstem/`). Bundled into every planting per Article XXIII.

**Documentation**: this vault note tells the rappter1 first-contact story as the canonical motivating example. Future readers should understand WHY substrate-agnostic federation became constitutional.

## The headline

**No centralized substrate is load-bearing for federation.** GitHub is convenient. LAN HTTP is a fallback. file:// is a fallback. USB-stick exchange is a fallback. The operator's brainstem on their device is the source of truth; the substrate is whatever URL serves the JSON.

The rappter1 case will not be the last. Operators will be GitHub-flagged; networks will be DPI'd; substrates will fail. The platform's local-first promise survives all of those failures. The federation walks whatever lights up.

That's the property the operator has been chasing this whole time: **a decentralized AI network that doesn't have a single point of failure — including the platform itself.**
