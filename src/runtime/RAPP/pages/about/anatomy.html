Anatomy of a Digital Organism

This diagram preserves the historical “living thing” metaphor. It is not an install, hatch, or shipped egg claim. Click a layer to inspect the source file associated with the former model.

Companion to the vault note Rapplications Are Organisms — collapsing a false distinction.

Kernel ↔ DNA

brainstem.py
The immutable historical runtime. Its three grail files are byte-pinned to kody-w/rapp-installer@brainstem-v0.6.9 and never edited locally; no current egg is accepted or hatched, and RAPP/1 convergence happens in target-owned adapters.

Soul ↔ Mind / personality

soul.md
The system prompt loaded on every chat turn. What makes BookFactory-the-organism different from a generic agent runtime. An organism at any scale can have its own soul.

Organs ↔ Internal musculature

utils/organs/*_organ.py
Single-file HTTP route handlers — name + handle(method, path, body) → (dict, status). Dispatched at /api/<name>/<path>. Organs serve UIs; agents serve LLMs. They never overlap.

Senses ↔ Perception channels

utils/senses/*_sense.py
Application output overlays for rendering. |||VOICE||| and |||TWIN||| are host conventions; overlays do not add RAPP wire fields or capability endpoints.

Cells ↔ Sub-organisms

agents/*_agent.py + hosted rapplications
Historical nested-organism concept. Eggification and transplantation are retired; any future identity or exchange must satisfy RAPP/1.

Memory ↔ Lived experience

.brainstem_data/
Application state accumulated by the organism. It travels only when explicitly listed in a valid §9 variant and remains independently subject to frame and trust verification.

Skin ↔ Visible outer layer EARNS THE TIER

.brainstem_data/rapp_ui/<id>/
The UI bundle — the organism's face, what the user sees and recognizes. The line that separates a bare agent from a graduated rapplication. A skinless organism is single-celled — useful, but only invokable through the host's mouth (chat). Skin lets the world point at it directly.

Egg ↔ Portable cartridge

RAPP/1 §9 organism egg (schema: rapp/1-egg)
One exact schema with six registered variants: organism, rapplication, session, invite, neighborhood, and estate. Carrier choice never changes the manifest or its verification.

Membrane ↔ Cell wall / process boundary

port + venv + filesystem scope
What separates one organism's territory from another's on the same device. Instance-scope organisms each get their own port; cell-scope sub-organisms share their host's membrane and live inside its process.

The recursion

The category is recursive: an organism contains organisms. A bare agent is single-celled. A rapplication is a cell that has grown skin. An entire brainstem instance is a body hosting many cells — its own DNA, its own soul, and a colony of sub-organisms inside. Same protocol at every scale.

"Skin is the criterion that makes 'rapplication' earn its name. A bare agent is a single-celled organism — internal, functional, but skinless. A rapplication has its own face — what the world sees and recognizes. That's why a rapplication needs more than a .py: a graduated organism requires skin."
