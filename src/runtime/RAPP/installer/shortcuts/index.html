Current boundary

Canonicalization, identity, frames, wire, eggs, registry, trust, and protocol evolution follow RAPP/1 rev-5 through RAPP1_AUTHORITY.json and RAPP1_STATUS.md.

✓ A future client sends only required string user_input plus optional strings session_id and idempotency_key.

✓ Success has exactly response, agent_logs (array), and session_id; HTTP 422 is exactly {"error":{"code":"<code>","step":null}}.

✓ Speech is derived locally from response. Voice and twin are not extra fields.

Private local experiment only

No Shortcut is offered for installation. If an operator deliberately authors a private experiment, it must use http://127.0.0.1:7073/chat, read response, and derive speech locally without adding wire fields.

  1. Read the exact request/response/error contract.
  2. Do not send history, mode flags, identity fields, or voice/twin fields.
  3. Do not publish or advertise the experiment as RAPP/1 conformant.
