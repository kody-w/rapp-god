# Eight-strategy cycle 2: prove who proposed what

The second council converged on provenance, council observability, and a hardened proposal boundary.

Evolution receipts are now score-replayed and hash-chained. Tampering with a score, bar, transition, or
event order fails closed. `warehouse/lineages.json` materializes ancestry, active state, component scores,
receipt hashes, model attribution, and floor evidence; the new **Evolution Observatory** presents that
lineage beside the product council's decisions.

The repository now carries an exact-eight council engine. Every strategy receives the same immutable
snapshot in a fresh no-tools Copilot CLI invocation, returns a strict five-proposal ballot, and participates
in deterministic support/priority/Borda consensus. Missing or malformed ballots abort the cycle instead of
quietly adding a ninth vote. Structured, content-addressed cycle receipts are append-only.

The proposal worker no longer sends untrusted title or author text to the model. It requests keyframes
only, echoes a revision-bound challenge ID, restores identity deterministically, disables tools and built-in
MCPs, removes credential-like environment variables, and rejects trailing output. An isolated live check
used the existing `gh copilot` login and produced a **0.5318** child against a **0.2468** bar.

Seventeen deterministic tests now cover validation, active-frontier progress, receipt tamper detection,
agent challenge/propose behavior, exact-eight consensus, ingestion authorization, and Python/browser
fitness parity.
