# 👁️ rapp-god

**The assimilated public RAPP monorepo and append-only registry of every
version of every part it is made of.**

This repository now dispositions the frozen public RAPP source census brick by
brick in semantic domains while keeping its original observatory and raw-URL
API intact. Privacy-sensitive references from public sources are withheld
opaquely rather than republished. Start with [`MONOREPO.md`](MONOREPO.md) for
the architecture, authority order, provenance, licensing boundary, update
process, and honest limits. The imported RAPP baseline is **NOT YET FULLY
RAPP/1 CONFORMANT**; four owner-action blockers remain open and no structural
hash is claimed as authenticated section-13 acceptance.

[RAR](https://github.com/kody-w/RAR) is a registry of agents. **rapp-god is a registry of the whole
ecosystem**: the kernel, the installer, the spec, the codec, the seed agents — every load-bearing
part. For each part it keeps every captured version as a content-addressed
frame. Explicit security purges retain tombstone metadata without restoring
the payload.

## The idea

- **Every captured version is a load-bearing fallback.** Each retained version of a part lives at its
  own raw URL (`versions/<part>/<sha8>`). If a canonical source breaks, vanishes, or ships a bad
  update, you pin to any prior frame and keep going. Not just the current version — *all* of them.
- **Update detection, never coercion.** Actionable updates come only from
  non-observe sources. Moving production-head mirrors remain informational.
- **It observes; it never fixes.** A fork doesn't get auto-reconciled — a drifted copy might be the
  better one. rapp-god shows you *exactly what drifted* (hash + inline diff) and leaves the call to you.
- **Fully static.** No server. The dashboard reads `registry.json`; everything is served globally
  over `raw.githubusercontent.com`.

## The static API — built entirely on raw.githubusercontent.com

No backend. Fetch these from anywhere:

| Endpoint | What |
|---|---|
| [`registry.json`](https://raw.githubusercontent.com/kody-w/rapp-god/main/registry.json) | all captured part versions, source assignments, and hash-verifiable fallback URLs |
| [`api/v1/status.json`](https://raw.githubusercontent.com/kody-w/rapp-god/main/api/v1/status.json) | the latest verdict (summary + per-part drift) |
| [`api/v1/badge.json`](https://raw.githubusercontent.com/kody-w/rapp-god/main/api/v1/badge.json) | shields.io endpoint badge |
| [`api/v1/ecosystem-spec.json`](api/v1/ecosystem-spec.json) | fail-closed quarantined candidate; **not** a RAPP/1 registry |
| `versions/<part>/<sha8>` | content-addressed, hash-verifiable fallback bytes; append-only by policy |

Raw `main` URLs are convenience transport, not transport-immutable objects.
Verify the recorded SHA-256 and content-addressed path before use.

**Check for an update** (stay or take it — your call):
```bash
GRAIL=$(curl -s https://raw.githubusercontent.com/kody-w/rapp-god/main/registry.json \
  | jq -r '.parts[]|select(.name=="install.sh").grail_sha8')
MINE=$(shasum -a256 install.sh | cut -c1-12)
[ "$GRAIL" = "$MINE" ] && echo "up to date" || echo "an update is waiting (grail $GRAIL)"
```

**Pin a fallback** — fetch one exact, immutable version:
```bash
curl -O https://raw.githubusercontent.com/kody-w/rapp-god/main/versions/install.sh/<sha8>.sh
```

## Build it

Like RAR: `manifest.json` is the hand-authored input, `build_god.py` is the only build step,
`registry.json` is generated (never hand-edited).

```bash
python3 build_god.py        # fetch sources + capped file history, capture frames, regenerate registry + API
python3 build_god.py --no-net   # regenerate from already-captured frames only
python3 build_god.py --check    # fail if offline generated outputs differ
```

`manifest.json` lists the **parts** — each with a `grail` (the source of truth) and its `mirrors`,
plus `"history": true` to inspect at most `RAPP_GOD_HISTORY_CAP` file-touching
commits (default 60), and `"kind"`:
- **`observe`** (default) — record drift, never fail CI. The drifted copy might be the keeper.
- **`enforce`** — fail CI on drift. Opt-in, per part, only once you've decided it must never differ.

CI (`.github/workflows/god-build.yml`) runs `build_god.py` on every push and every 6h, commits any
new frames + registry changes (and *only* when something actually changed), and **stays green for
observed drift**. A mismatch in the pinned LTS authority is an integrity failure.

## Staging the assimilated tree

Nested imported `.gitignore` files intentionally remain exact. The initial
plan found 985 ignored paths; private-boundary withholding leaves 983 retained
ignored paths. Never stage this monorepo with a plain `git add`. After
privacy/quarantine and integrity checks pass, use the
target-owned NUL-safe closure tool:

```bash
python3 tools/stage_materialized.py --plan
python3 tools/stage_materialized.py --stage
python3 tools/stage_materialized.py --check
```

It force-stages ledger destinations and target-owned files, passes NFC logical
paths to Git on macOS, and verifies every index path/blob/mode. It does not
commit or push.

## The grail pattern

RAPP's own canon calls the frozen kernel a **grail**, and instances "fetch from grail." The enforced
authority here is the exact `rapp-installer@brainstem-v0.6.9` tree at commit `bded0e1`; moving
`rapp-installer/main` is retained as an observe-only upstream and cannot replace that pin. Historical
RAPP planting checks against main remain visible as implementation evidence. rapp-god generalizes
the grail-diff into a **public, global, per-file, every-version** registry.
