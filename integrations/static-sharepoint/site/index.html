Static SharePoint

Deterministic document-collaboration fixtures for the fictional tenant Aster Lane Office Systems, served as classic SharePoint-style REST shapes ({"d": {"results": [...]}}) from plain static JSON.

Independently authored simulator serving SharePoint-style REST shapes. Not affiliated with, endorsed by, or claiming parity with Microsoft. All content is synthetic: customer domains use .example and no real organizations, people, or documents are represented. Document binaries deliberately do not exist — each item carries a one-paragraph Preview instead (an enrichment seam).

Lists

ListItemsDescription
Contracts14Executed and in-flight customer agreements: MSAs, SOWs, and renewals.
Policies8Internal HR, benefits, finance, security, and compliance policies.
Meeting Notes10Customer and internal meeting notes referencing CRM cases and opportunities.

_api/web/lists.json — index of all lists with item counts.

Read example

const response = await fetch("./_api/web/lists/Contracts/items.json");
const contracts = await response.json();
console.log(contracts.d.results.length, contracts.d.results[0].Title);

Cross-system joins

Contracts and meeting notes reference CRM case ticket numbers (RelatedTicket, e.g. CAS-260134) and shared customer accounts (RelatedAccount) from the companion Static Dynamics 365 tenant, so an agent can join documents to live-shaped CRM records.

Write API

Open a GitHub Issue titled [SP] ... carrying a fenced sharepoint-write/1.0 JSON command to create Contracts or Meeting Notes items, or update a contract Status by Id. See the repository for the command shapes.
