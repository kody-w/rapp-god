OpenRappter Release Train

Every ring is its own repo and its own npm package. Production is unreachable from ring work.

The train

Promotion republishes the identical tarball under the next ring's package name. It never rebuilds — the bytes that enter at canary are the bytes that reach stable. Only a real release ever writes openrappter.

Install a ring

RingPackageRepoInstaller

Switch rings after install

openrappter --version                 # what you're on
curl -fsSL https://kody-w.github.io/openrappter/install.sh | bash -s -- --channel beta

Your ring is remembered in ~/.openrappter/channel, so later upgrades stay on the same train car. A version pin applies within the selected ring — OPENRAPPTER_VERSION=1.9.8 with --channel canary resolves openrappter-canary@1.9.8, never the production package.
