# Roadmap

Feature priorities and upcoming work for openrappter.

## Current State (v1.9.3)

### Completed
- [x] Dual runtime: TypeScript + Python with parity
- [x] 26+ TypeScript agents, 20+ Python agents
- [x] 18+ messaging channels
- [x] 5 LLM providers (Copilot, Anthropic, OpenAI, Gemini, Ollama)
- [x] Data sloshing (implicit context enrichment)
- [x] Data slush (agent-to-agent pipeline)
- [x] Multi-agent composition (Chain, Graph, Router, Broadcast, SubAgent, Pipeline)
- [x] AgentTracer (span-based observability)
- [x] MCP server (Claude Code / Cursor integration)
- [x] Dashboard REST API + Lit web UI
- [x] 20 showcase demos with tests
- [x] ClawHub + RappterHub skill marketplace
- [x] macOS menu bar app (dino tamagotchi)
- [x] Background daemon (launchd/systemd)
- [x] Self-healing cron agent
- [x] Self-evolving agents (Ouroboros + Watchmaker)
- [x] 30-day onboarding (DailyTipAgent)
- [x] Soul templates (10 personas)
- [x] Backup & restore system
- [x] LispyCoach (LLM-generated strategies)

### In Progress
- [ ] Productivity stack automation (see [[Productivity Stack Plan]])
- [ ] Integration testing with real external services

## Potential Next Steps

### Short-term
- Real GitHub/GitLab integration for code monitoring
- Email triage (Gmail OAuth / IMAP)
- Project board connectors (Jira, Linear, Trello)
- Enhanced memory with vector embeddings

### Medium-term
- Multi-user / team support
- Plugin system for third-party extensions
- Mobile companion app
- Agent marketplace with ratings/reviews

### Long-term
- Federated agent networks
- Cross-runtime agent interop (TS agents calling PY agents)
- Enterprise deployment (SSO, audit logs, compliance)
