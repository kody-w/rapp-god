WHY THIS RELEASE MATTERS

Power is only useful when the whole system holds together.

🛡️

Execution is gated, not merely labeled safe

Shell commands, dual-use tools, generated code, ComputerUse, and recorder process boundaries now fail closed with exact, single-use approvals and argv-safe execution.

🧠

Python became a durable runtime

SQLite restart persistence, migrations, bounded agent workers, a real HTTP/WebSocket gateway, providers, channels, and wheel-installed socket tests replace parity claims with running behavior.

🔐

One authenticated RPC contract

HTTP and WebSocket calls share fail-closed auth, canonical channel/session names, real streaming frames, stable errors, and compatibility aliases for web and Swift clients.

📈

The system can explain itself

Bounded counters, health/status metrics, redacted JSON logs, real socket tests, and deterministic native lifecycle tests make failures observable instead of mysterious.

🍎

The Mac app owns its lifecycle

Connection generations, idempotent shutdown, exact-PID escalation, external-process protection, and deterministic tests prevent stale sockets and orphaned gateways.

🚀

A release now means one exact artifact set

One fail-hard OIDC release lane validates versions, builds once, smoke-installs exact npm/Python artifacts, reconciles registry digests for safe retries, and protects npm latest with semantic-version ordering.

TRY THE POWER

10 prompts that make OpenRappter feel impossible.

Paste one into your agent. Every prompt keeps mutations and external sends behind explicit approval.

1. The Autonomous CTO

Audit this product through eight independent strategy lenses. Build a ranked consensus, implement the top three compatible improvements, validate every runtime, independently review the diff, and publish a release note explaining what changed and why. Checkpoint compact state after every phase.

2. The Time-Travel Debugger

Reproduce this intermittent bug, then fan out competing causal hypotheses across logs, traces, code history, race analysis, and protocol state. Build a timestamped execution narrative, eliminate hypotheses with experiments, fix the root cause, and add a regression test that fails if time ordering changes again.

3. Build My Digital Twin

Interview me to infer my priorities, working style, decision rules, recurring responsibilities, and communication preferences. Create a local soul, memory model, daily brief, and approval-gated automations. Show every inferred rule and let me accept, edit, or reject it before activation.

4. The Zero-Downtime Self-Healer

Turn this service into a self-healing system. Define health invariants, detect degraded states, trace the failure, attempt the safest repair, verify recovery, and escalate with evidence when repair fails. Simulate healthy, recoverable, and persistent-failure paths before enabling anything.

5. Darwin's Agent Colosseum

Create five radically different agents to solve this problem. Run them in parallel through hidden edge cases, have independent evaluators score correctness, safety, speed, and simplicity, evolve the weakest two, then synthesize a champion that preserves the best strategy from every contender.

6. The Cross-Runtime Mirror Test

Implement this capability independently in TypeScript and Python. Generate adversarial and property-based inputs, execute both runtimes in parallel, compare outputs, state transitions, errors, and performance, then resolve every semantic divergence without forcing one language into the other's idioms.

7. The Local-First War Room

Treat this incident description as a live war room. Inspect gateway health, active connections, traces, storage, channels, workers, and recent changes. Build a concise incident dashboard, identify blast radius, propose reversible mitigations, request approval for the safest one, execute it, and verify recovery.

8. The Skill Forge

Turn this plain-English capability into a single-file agent. Infer the minimal typed contract, generate deterministic implementation and tests, adversarially review its permissions and dependencies, quarantine it for inspection, request explicit approval, then hot-load it and demonstrate a real end-to-end call.

9. My Personal Operations Command Center

Using only my local memory, approved files, sessions, and connected channels, design a command center that produces a morning brief, identifies blocked work, drafts follow-ups, schedules recurring checks, and learns which signals I ignore. Never send or mutate externally without approval.

10. The Ship of Theseus Refactor

Modernize this application one subsystem at a time while keeping every observable behavior stable. Build contract tests first, replace internals behind compatibility boundaries, continuously compare old and new paths, visualize migration progress, and delete the legacy path only when parity is proven.

The raptor did not just get more features.

It gained the ability to improve itself without losing the thread: consensus, bounded context, explicit trust, executable proof.
