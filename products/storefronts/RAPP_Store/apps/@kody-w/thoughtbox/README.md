# @kody-w/thoughtbox

Local-first reflection journal. Drop the egg into any standard brainstem; the agent and UI auto-discover. Append a thought, search by text, filter by tag, export the whole thing as a portable JSON blob whenever you want it on another machine.

## Why

- **Local-first.** Nothing leaves the box. No telemetry, no central server, no API keys. The agent makes zero network calls.
- **Stateless agent.** All persistence routes through the workspace cartridge protocol (SPEC §12) — `entries.json` under the rapplication's per-id workspace key. No filesystem assumptions on the agent's side; the brainstem decides where state lives.
- **Portable.** The `export` action returns a single JSON blob with the schema `thoughtbox/1.0`. Drop it into another machine's Thoughtbox via `import_json` and you have your journal there too. The egg cartridge itself ships the agent + UI; your data is separate.

## Actions

| action | required args | does |
|---|---|---|
| `append` | `text` (`tags?` optional list) | Append a new entry. Tags inline with `#tagname` in text are auto-extracted by the UI. |
| `list` | (`limit?`) | Newest-first list. |
| `search` | `query` | Substring match on text. |
| `tag` | `tag` | Filter entries that carry this tag. |
| `stats` | — | Total count, earliest/latest timestamps, top tags. |
| `export` | — | Returns the full journal as a `thoughtbox/1.0` JSON blob. |
| `import_json` | `blob` | Merge entries from a JSON blob; existing IDs skip. |
| `delete` | `id` | Delete by full or short (8-char prefix) ID. |

## Loading the egg

```python
# brainstem.py side, with rapplication egg loader
from rapp_brainstem.bond import hatch_egg
hatch_egg("thoughtbox.egg")  # installs agent + UI route
```

Or via `rapp install` if your brainstem ships a CLI.

## Local development

The bundle lives at `apps/@kody-w/thoughtbox/`:

```
manifest.json              ← spec contract
index_entry.json           ← catalog row (sha256, urls, lines, bytes)
singleton/thoughtbox_agent.py
ui/index.html              ← cartridge-protocol UI (postMessage to parent)
eggs/thoughtbox-1.0.0.egg  ← portable cartridge for hand-distribution
README.md                  ← this file
```

To rebuild the egg after editing the singleton or UI:

```bash
cd apps/@kody-w/thoughtbox
python3 -m zipfile -c eggs/thoughtbox-1.0.0.egg \
    manifest.json singleton/thoughtbox_agent.py ui/index.html
```

(The published catalog egg at `api/v1/egg/thoughtbox.egg` is regenerated by `python3 scripts/build_pokedex_api.py`.)
