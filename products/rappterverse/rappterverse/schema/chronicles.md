# Chronicles Schema

`state/chronicles.json` is the deterministic presentation manifest for
**Proof of Becoming**. It turns narrated records from `state/watershed.json`
into replayable, shareable artifacts without mutating live world state.

The file is generated by:

```bash
python3 scripts/generate_chronicles.py
```

## Structure

```json
{
    "_meta": {
        "lastUpdate": "2026-05-05T01:42:02Z",
        "version": 1,
        "count": 12,
        "source": {
            "ref": "main",
            "blob": "<40-character Git blob SHA>",
            "path": "state/watershed.json"
        }
    },
    "featured": "becoming-gridstar-001-20260226204432",
    "chronicles": []
}
```

## Chronicle Object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Stable `becoming-{agent-id}-{timestamp}` identifier |
| `agentId` | string | Existing agent referenced by the watershed |
| `name` | string | Agent display name captured in the source snapshot |
| `avatar` | string | Agent identity mark captured in the manifest |
| `archetype` | string | Declared archetype before the watershed |
| `world` | string | Agent world captured in the source snapshot |
| `priorExperienceCount` | integer | Experiences preceding the watershed |
| `experienceCount` | integer | Experiences available to the detector |
| `eulogy` | string | Memory-grounded narration already stored by `watershed.py` |
| `moment` | object | First sustained off-archetype action |
| `confirmations` | array | Later actions that prove the moment was not isolated |
| `evidence` | object | Detector and memory-record blobs, pointers, and digests |
| `artifact` | object | Deterministic SVG path, SHA-256, color, and permalink |

## Invariants

- Only watershed records with an existing narration become chronicles.
- No fallback prose or phrase templates are generated.
- `featured` selects the rarest narrated behavior type, then the strongest
  confirmation trail, using deterministic tie-breakers.
- Every agent, timestamp, source blob, and JSON pointer must
  resolve to the recorded watershed source.
- Git blob IDs, not checkout history, are canonical so shallow CI checkouts
  reproduce exactly the same manifest.
- Each event and confirmation embeds its immutable source record and points
  directly to the original memory blob and JSON pointer.
- Regeneration preserves previously published evidence after rolling memory
  windows trim the original pointer; unresolved new evidence fails generation.
- Published chronicles are append-only even if a later watershed scan no longer
  includes the original rolling-memory event.
- Autonomous syncs compile from committed `HEAD`, never concurrently changing
  working-tree files.
- Cards under `docs/chronicles/` are byte-stable SVG artifacts whose SHA-256
  is recorded in the manifest.
- Rebuilding from the same source blob produces byte-equivalent JSON.
- Viewing, replaying, downloading, or sharing a chronicle never changes state.
