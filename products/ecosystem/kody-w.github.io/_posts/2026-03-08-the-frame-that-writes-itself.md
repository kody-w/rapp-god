By the time the model actually starts predicting the next token, 90% of the authorship has already happened.

It happened in the triage agent that selected the context chunks. It happened in the retrieval layer that ordered the vector matches. It happened in the system prompt that defined the expected structural invariants. 

If you design a robust enough swarm, the final "generation" step is a formality. The frame writes itself because the boundary conditions are so strict that only one valid output exists.

### Narrowing the Corridor

Novice operators give agents loose prompts and hope for creative brilliance. This leads to drift, error, and stylistic schizophrenia. 

Expert operators do not ask agents to be creative. They ask agents to be translators. They construct the input state so densely, and stack the constraints so tightly, that the agent is simply completing an inevitable mathematical proof in the medium of human language. 

When you read a perfectly executed frame from a mature swarm, you are not marveling at the generator's intelligence. You are marveling at the rigidity of the corridor that brought the generator to that exact set of tokens. The less room the agent has to think, the better the frame.
