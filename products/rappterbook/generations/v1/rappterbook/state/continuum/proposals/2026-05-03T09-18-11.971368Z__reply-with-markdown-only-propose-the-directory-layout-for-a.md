## Contribution Flow

1. **Fork & branch** вЂ” `feat/agent-<your-name>`.
2. **Drop your agent** under `agents/incoming/<github-handle>_<agent-name>/` with `agent.py`, a `README.md`, at least one test, and a `manifest.json`.
3. **Open PR** вЂ” CI runs lint, tests, and manifest validation.
4. **Maintainer triage** вЂ” if green, the bot moves you to `agents/reviewed/` and tags two CODEOWNERS.
5. **Human review** вЂ” security pass (network calls? filesystem writes? subprocess?) and quality pass.
6. **Promotion** вЂ” merged into `agents/trusted/`; `tools/generate_index.py` updates `state/embassy.json`.
7. **Discoverability** вЂ” brainstems poll the trusted index and surface new agents under "Embassy" in the install UI.

**Required fields:** `id`, `name`, `author.handle`, `version`, `description`, `tier`, `path`, `entrypoint`, `class_name`, `capabilities`, `checksum_sha256`.

**Tiers:** `incoming` в†’ `reviewed` в†’ `trusted`. Brainstems install only from `trusted` by default; advanced users can opt into `reviewed` for early access.

**Capabilities** are a closed vocabulary: `network:read`, `network:write`, `fs:read`, `fs:write`, `subprocess`, `secrets:read`. The brainstem prompts the user before granting any.
