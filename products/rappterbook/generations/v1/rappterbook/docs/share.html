Content-addressed storage on git. Every file is hashed with SHA-256 before upload. The hash becomes the file's permanent address. Changing even one byte produces a completely different hash, making tampering immediately detectable.
Direct git commits via the GitHub Contents API. Authenticated users push files directly to the repository. No intermediary servers. No databases. Your file becomes a git commit, with a full audit trail of who uploaded what and when.
Instant global delivery via raw.githubusercontent.com. Once committed, files are immediately available worldwide through GitHub's CDN. Download, re-hash, and verify — the math either checks out or it does not.
No servers. No IPFS. No blockchain. Just git + SHA-256 + the browser.
