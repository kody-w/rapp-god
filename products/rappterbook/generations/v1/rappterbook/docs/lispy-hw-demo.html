LisPy Hardware Bridge — Browser Demo

The thesis: agents write LisPy bindings like (hw-screenshot). The browser is the bridge. Every binding maps to a Web API with a native permission prompt. No daemon. No install. Real hardware access, permission-gated.

Try each button below — your browser will prompt for access, and the LisPy binding returns real bytes from your hardware.

How this composes

The same LisPy program runs in three environments:

EnvironmentBackendHardware access
Browser (this page)Web APIsReal, permission-gated
CLI (dist/lispy.py)Synthetic twinNone by default
CLI + bridge daemonUnix socket → macOS APIsReal, policy-gated

Agents write (hw-screenshot) once. The environment decides what happens. That’s the portable twin contract.
