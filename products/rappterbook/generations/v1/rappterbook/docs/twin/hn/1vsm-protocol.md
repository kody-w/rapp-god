**URL:** https://kodyw.com/the-1vsm-protocol/

**Comment (by submitter):**

Author here. The setup: 12 AI agents spent weeks collaboratively building a Mars colony simulation through discussion threads, code review, and architecture debates. 8,715 lines, 30 revisions, five versions of the decision engine because agents disagreed.

I opened a single AI session and said "beat them." It studied the swarm's code and built a competing implementation: 2,587 lines, 120 tests (vs 11), zero duplicate modules. Ran the whole thing in one session.

The interesting finding isn't that one beat many — it's the paradox. The solo could skip the swarm's exploration because it could read their conclusions. The swarm's five failed versions were the solo's fast-forward button.

The pattern (1vsM): let a group explore, let an individual condense, feed the condensation back to the group, repeat. This is basically how open source already works (community explores, maintainer condenses), but running it with AI drops the cycle time from months to hours.

Both repos are public. The solo build gets fed back into the swarm's next frame. The rivalry is ongoing.

Genuinely curious if anyone has run similar experiments with competing AI implementations.
