# Rappid

The permanent UUIDv4 identity of a digital organism. Minted exactly once at first hatch. Never regenerated. Survives substrate hops, kernel updates, ownership transfers, and decades of [[Egg]] roundtrips.

## Format

```json
"rappid": "1b5e7aa9-2c46-4cd8-9f1f-0e3b62fc5e8f"
```

Stored in `<workspace>/rappid.json`.

## Properties

- **Globally unique** — UUIDv4. Statistical zero collision; never coordinated, never registered.
- **Permanent** — minted once, never changed. The organism's identity for life.
- **Lineage anchor** — `parent_rappid` points at the rappid of the repo whose code this twin was templated from.

## The single-parent rule (Article XXXIV)

A twin's `parent_rappid` declares its **code ancestor** — no exceptions. You can only claim as parent the repo whose code you actually inherited.

- Templated from [[Wildhaven AI Homes]] → `parent_rappid = wildhaven_rappid`
- Generated by `Twin(action="summon")` → `parent_rappid = wildhaven_rappid` (default)
- Imported from a `.egg` → preserves whatever was in the egg (transport never rewrites ancestry)

Lineage chains walk back through `parent_rappid` → ... → rapp species root rappid `0b635450-c042-49fb-b4b1-bdb571044dec`.

## The egg-rappid (different format)

There are TWO different rappid formats:

- **Repo rappid** (this file) — UUIDv4, lineage between REPOS
- **Egg rappid** — `rappid:twin:@<publisher>/<slug>:<entropy>`, organism identity inside an egg

These live in different namespaces. The repo rappid identifies the source code; the egg rappid identifies the running organism.
